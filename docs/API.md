# API Reference — ms-gateway-llm

Production endpoint reference. The gateway exposes two surfaces:

- **Public surface** (`/v1/...`) — used by LLM clients (Kilo, OpenCode, Claude Code, custom integrations). Every endpoint requires a valid API key. Model identity is hidden behind aliases.
- **Admin surface** (`/admin/...`) — operator tooling for client CRUD, alias routing policy, and request-log inspection. Requires `admin` scope in addition to a valid API key.

> **Defense-in-depth**: the public surface (`/v1/models`, `/v1/metrics/summary`) only ever exposes alias names. Admin endpoints (`/admin/logs`) DO surface `resolvedProvider` / `resolvedModel` because operators need it for incident triage — gated behind the `admin` scope. Internal logs (stdout / NewRelic) still see all of it.

---

## Authentication

### Bearer token (recommended)

```http
Authorization: Bearer sk-<your-key>
```

### Alternate header

```http
X-API-Key: sk-<your-key>
```

Notes:

- The key is the plaintext generated by the admin CLI on client creation or rotation. It is never returned by the API after creation.
- Keys are stored as `HMAC-SHA256(API_KEY_PEPPER, plaintext)` with a server-side pepper. A leaked DB alone is **not** enough to forge a verification.
- Per-request validation is cached in Redis for 5 minutes (`ak:v1:<prefix>:<sha256(plaintext)>`). Rotations / revocations propagate within that window.

## Public surface (`/v1/...`)

### `GET /v1/models`

List the model aliases the gateway exposes.

**Auth**: required. **Rate-limit**: yes (per client RPM).

**Response** (OpenAI-compatible):

```json
{
  "object": "list",
  "data": [
    {
      "id": "coder",
      "object": "model",
      "created": 1783350932,
      "owned_by": "nan"
    }
  ]
}
```

Only the alias `id` is ever returned. Real upstream model ids (`qwen3.6`, `deepseek-v4-flash`, etc.) are **never** exposed. Aliases are advertised only if their primary chain entry points to a provider with the API key env configured and the upstream model key exists.

---

### `POST /v1/chat/completions`

OpenAI-compatible chat completions. Supports both buffered and SSE streaming.

**Auth**: required. **Rate-limit**: yes.

**Request body**:

```json
{
  "model": "coder",
  "messages": [
    {"role": "system", "content": "You are a helpful coding assistant."},
    {"role": "user", "content": "Write a quicksort in Python."}
  ],
  "stream": false,
  "temperature": 0.2,
  "max_tokens": 1024
}
```

Required: `model` (one of the aliases from `/v1/models`), `messages` (≥1).
Common optional: `stream`, `temperature`, `top_p`, `n`, `max_tokens`, `presence_penalty`, `frequency_penalty`, `seed`, `user`, `stop`, `tools`, `response_format`.
Passthrough: the schema is lenient — unknown fields are forwarded to the upstream SDK.

**Streaming** (`"stream": true`):
- Content-Type: `text/event-stream`
- Body: `data: {<chunk>}\n\n` chunks, terminated with `data: [DONE]\n\n`
- Errors mid-stream are emitted as `data: {"error": {"message": "..."}}\n\n` then `[DONE]`.

**Buffered response** (`"stream": false`, default): standard OpenAI `ChatCompletion` object (`id`, `object: "chat.completion"`, `created`, `model`, `choices`, `usage`).

**Errors**:

| Status | Meaning |
|---|---|
| `400` | Body failed zod validation |
| `401` | Missing or invalid API key |
| `429` | Rate limit exceeded; `Retry-After` header carries seconds |
| `500` | Upstream or gateway error; `{"error": {"message": "..."}}` |

---

### `GET /v1/health`

Process health check. **No auth required.** `@HealthCheck()` from `@nestjs/terminus`.

```json
{
  "status": "ok",
  "info": {
    "memory_heap": {"status": "up"},
    "memory_rss": {"status": "up"}
  },
  "error": {},
  "details": {
    "memory_heap": {"status": "up"},
    "memory_rss": {"status": "up"}
  }
}
```

Heap limit: 400 MiB. RSS limit: 500 MiB. Either breached → `status: "error"`.

---

### `GET /v1/health/llm`

Per-provider gateway health snapshot. **No auth required.**

```json
{
  "status": "ok",
  "providers": [
    {
      "id": "nan",
      "state": "closed",
      "failures": 0,
      "canServe": true,
      "lastFailureAt": 1783350000,
      "openedAt": 1783350000
    }
  ]
}
```

Aggregate `status`:

- `"ok"` — every provider can serve.
- `"degraded"` — at least one provider can serve.
- `"down"` — no provider can serve.

`state` is the circuit-breaker state (`closed` / `open` / `half-open`).

---

### `GET /v1/metrics/summary`

Aggregated metrics over a rolling window. **No auth required.**

Query parameters:

| Param | Type | Default | Notes |
|---|---|---|---|
| `window` | `1h` \| `24h` \| `7d` | `1h` | Bad value → `400` |
| `now` | integer (unix seconds) | server time | Non-numeric / negative → `400` |

Response:

```json
{
  "window": "1h",
  "since": 1783349400,
  "until": 1783353000,
  "totals": {
    "requests": 184,
    "errors": 3,
    "error_rate": 0.016
  },
  "models": [
    {
      "model": "coder",
      "requests": 184,
      "errors": 3,
      "error_rate": 0.016,
      "latency_ms": {"p50": 4200, "p95": 12000, "p99": 18000, "min": 350, "max": 22000}
    }
  ]
}
```

Aggregation is by alias only (`model_requested`). When a single alias routes to multiple upstream providers via its fallback chain, those rows are merged into a single entry — the response carries no per-provider dimension. Real provider / model identity stays inside the gateway (server stdout via `LlmLoggingService`, plus direct SQL against `request_logs`).

## Admin surface (`/admin/...`)

Every admin endpoint requires:

- A valid API key.
- The `admin` scope on the client (configured at create time; see `POST /admin/clients`).
- Standard per-client rate-limit.

Status codes follow REST conventions. Error envelope: `{"error": {"message": "..."}}` (NestJS default).

---

### `GET /admin/clients`

List all clients (including revoked).

```json
{
  "clients": [
    {
      "id": "admin",
      "name": "Admin",
      "scopes": ["admin", "chat.read", "chat.write"],
      "rateLimitRpm": 1000,
      "rateLimitTpm": null,
      "apiKeyPrefix": "sk-aaaaab",
      "createdAt": 1780000000,
      "lastUsedAt": 1783353000,
      "revoked": false
    }
  ]
}
```

---

### `GET /admin/clients/:id`

Get one client. `404` if id unknown.

---

### `POST /admin/clients`

Create a new client.

**Request**:

```json
{
  "id": "tenant-acme",
  "name": "Acme Co.",
  "scopes": ["chat.read", "chat.write"],
  "rateLimitRpm": 300,
  "rateLimitTpm": 100000
}
```

| Field | Required | Notes |
|---|---|---|
| `id` | no (auto-generated slug if absent) | `^[a-z0-9][a-z0-9_-]*$`, max 64 chars |
| `name` | yes | 1–120 chars |
| `scopes` | no | defaults `["chat.read","chat.write"]`. Must be non-empty. |
| `rateLimitRpm` | no | default `60`. Must be > 0. Max `100_000`. |
| `rateLimitTpm` | no | default `null` (no token cap). Max `100_000_000`. |

**Response** (`201`):

```json
{
  "id": "tenant-acme",
  "name": "Acme Co.",
  "scopes": ["chat.read", "chat.write"],
  "rateLimitRpm": 300,
  "rateLimitTpm": 100000,
  "apiKeyPrefix": "sk-aaaaab",
  "createdAt": 1783353000,
  "lastUsedAt": null,
  "revoked": false,
  "plaintextApiKey": "sk-…",
  "warning": "Save this API key now. It will never be shown again."
}
```

The `plaintextApiKey` is returned **once**, only here.

---

### `PATCH /admin/clients/:id`

Partial update.

```json
{
  "name": "Acme (renamed)",
  "scopes": ["chat.read"],
  "rateLimitRpm": 500,
  "rateLimitTpm": null
}
```

All fields optional. Pass `rateLimitTpm: null` to clear the token cap.

---

### `POST /admin/clients/:id/rotate`

Mint a new API key for an existing client. Same response shape as `POST /admin/clients`, including the **new** `plaintextApiKey`.

**`404` if id unknown. `400` if client is revoked.**

---

### `POST /admin/clients/:id/revoke`

Mark a client revoked. `204` on success. The cache takes up to 5 minutes to clear the entry.

---

### `DELETE /admin/clients/:id`

Hard-delete a client row. `204` on success (idempotent — `204` also if id unknown).

---

### `GET /admin/aliases`

List all alias routing chains.

```json
{
  "aliases": [
    {
      "id": "coder",
      "chain": ["nan/qwen3.6", "nan/deepseek-v4-flash"],
      "strategy": "primary",
      "weights": [1, 1],
      "priorities": [0, 1]
    }
  ]
}
```

`chain` is the ordered fallback list. `strategy` is `primary` / `round-robin` / `weighted` / `priority`. `weights` and `priorities` are 1:1 with `chain` positions.

---

### `GET /admin/aliases/:id`

Single alias. `404` if id unknown.

---

### `PUT /admin/aliases/:id/strategy`

Per-alias routing strategy.

```json
{"strategy": "weighted"}
```

Accepted values: `primary`, `round-robin`, `weighted`, `priority`. `204` on success.

---

### `PUT /admin/aliases/:id/weights`

Per-position weights. **Length must match the chain length.**

```json
{"weights": [1, 3]}
```

`204` on success. `400` if `weights.length !== chain.length`.

---

### `PUT /admin/aliases/:id/priorities`

Sparse per-position priority map. Positions not listed keep their current value.

```json
{"priorities": {"0": 5}}
```

`204` on success. `400` if any listed position is out of `[0, chain.length - 1]`.

---

### `GET /admin/logs`

Recent request_logs with filters. **Admin-privileged** — unlike the public surface, this endpoint exposes `resolvedProvider` and `resolvedModel` so operators can see which upstream actually served each request.

Query parameters (all optional, AND-combined):

| Param | Type | Notes |
|---|---|---|
| `client_id` | string 1–64 | Matches `client_key` |
| `model` | string 1–120 | **Alias**, e.g. `code`. |
| `provider` | string 1–64 | Filters on `resolved_provider` (admin sees it). |
| `status` | `ok` \| `error` \| `circuit_open` | |
| `from` | ISO-8601 datetime | Inclusive lower bound on `requested_at` |
| `to` | ISO-8601 datetime | Inclusive upper bound on `requested_at` |
| `limit` | integer 1–500 | Default `100` |

`403` if the request comes from a client without `admin` scope.

**Response**:

```json
{
  "items": [
    {
      "requestedAt": 1783353000,
      "modelRequested": "code",
      "resolvedProvider": "nan",
      "resolvedModel": "qwen3-coder",
      "attempts": 1,
      "latencyMs": 4200,
      "status": "ok",
      "error": null,
      "clientKey": "tenant-acme",
      "promptHash": "26296f7c…",
      "promptTokens": 70441,
      "completionTokens": 86,
      "totalTokens": 70527
    }
  ],
  "count": 1,
  "limit": 100,
  "hasMore": false
}
```

- Newest first, ordered by `(requestedAt DESC, id DESC)`.
- `hasMore: true` means at least one row matched beyond `items`.
- Indexes used: `(client_key, requested_at DESC, id DESC)` and `(status, requested_at DESC, id DESC)`. `?model=` and `?provider=` are `requested_at` scans, fine while volume stays ≤ ~5M rows.
- `resolvedProvider` / `resolvedModel` are exposed here **on purpose** — admin scope is the privilege boundary. The public surface (`/v1/models`, `/v1/metrics/summary`) still hides them.

## Status codes (cross-cutting)

| Status | When |
|---|---|
| `200` | Successful non-empty response |
| `201` | Resource created (POST /admin/clients) |
| `204` | Successful mutation with no body (POST /:id/revoke, PUT /admin/aliases/..., DELETE) |
| `400` | Body or query failed validation (zod) — `{"error": {"message": "...", "issues": [...]}}` |
| `401` | Missing / invalid API key, no client attached |
| `403` | Client authenticated but missing required scope |
| `404` | Resource id unknown |
| `429` | Rate limit exceeded. `Retry-After: <seconds>` header set. `{"statusCode":429,"error":"Too Many Requests","message":"...","retryAfterMs":...}` |
| `500` | Upstream error or unhandled gateway error — `{"error": {"message": "..."}}` |

Streaming `chat/completions` errors: errors are emitted as `data: {"error":{"message":"..."}}\n\n` followed by `data: [DONE]\n\n`. The HTTP status is `200` even on streaming errors (chunk-by-chunk).

## Provisioning flow (operators)

Creating an API key for a developer who already has one (POC migration path):

```bash
# 1. Generate the pepper once and store it in Doppler / `.env`.
export API_KEY_PEPPER=$(openssl rand -hex 32)

# 2. Hash the existing plaintext key for DB insertion.
pnpm admin:reset -- --reset <client-id> --plain "<existing-key>"

# 3. Paste the emitted `UPDATE clients SET …` SQL into the proxy's DB.
sqlite3 data/ms-gateway.db < <(pnpm admin:reset -- --reset <client-id> --plain "<existing-key>" | sed -n '/UPDATE/,/;$/p')
```

See `docs/API-KEYS.md` for the longer walkthrough.

## What's NOT exposed

- **`api_key_hash`** — never returned. Hashes live in the DB; the API only ever returns `apiKeyPrefix`.
- **Upstream model ids** (`qwen3.6`, `deepseek-v4-flash`, etc.) — never returned by `/v1/models` or `/v1/metrics/summary`. Aggregation in `/v1/metrics/summary` is per-alias only; per-provider totals are not exposed. **Admin endpoints** (`/admin/logs`) DO include them, behind the `admin` scope.
- **Pepper / secret material** — never returned, never logged.
- **Cache contents** — internal only; no admin endpoint exposes Redis state.
